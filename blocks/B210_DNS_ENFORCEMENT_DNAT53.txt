BLOCK-ID: B210
TITLE: DNS Enforcement (DNAT 53)
VERSION: v1.7
STATUS: ACTIVE
PRIORITY: MUST
OWNER-INTENT: "Idiotensicher" DNS egal was User einträgt.
================================================================================

1) ZWECK
- Redirect TCP/UDP 53 from ppp* to 10.77.0.1:53.
- Gilt für ALLE ppp* Clients (User/Admin/restricted), damit DNS immer über AdGuardHome läuft.

2) GRENZEN
- Aktivieren erst wenn DNS stack stabil (B200).
- Dieses Enforcement betrifft NUR klassisches DNS über UDP/TCP Port 53.
  - DoH (HTTPS/443), DoT (TCP/853) und DoQ sind NICHT Teil dieses Blocks.
  - Falls später ein "Total-DNS-Lockdown" gewünscht ist, ist das eine separate Outbound-Policy-Entscheidung (eigener Block/Decision).
- IPv4-only: Dieser Block beschreibt DNS-Enforcement für IPv4.
  - Konzept-Policy: IPv6 ist serverseitig deaktiviert/gedroppt (Stealth).
  - Falls IPv6 jemals aktiviert wird, muss DNS-Enforcement für IPv6 explizit ergänzt werden, sonst potentieller DNS-Leak über v6.
- DNAT betrifft nur eingehenden Client-Traffic auf iifname "ppp*".
  - Server-eigene DNS Queries (OUTPUT) sind nicht Teil dieses DNAT und werden über lokale Resolver-Config gesteuert (z.B. Unbound).

3) ABHÄNGIGKEITEN
- Requires: B200, B120, B150, docs/nft_service_ip_scopes.txt

4) INPUTS
- nft nat prerouting for interfaces ppp*

5) OUTPUTS
- forced DNS
- Empfehlung (Kanon): Server nutzt Unbound direkt als Resolver (127.0.0.1:5335) statt AdGuard, um Abhängigkeiten/Loops zu minimieren; Client-DNS bleibt via DNAT immer AdGuard.

6) MUST (wenn aktiv)
- Both UDP and TCP 53
- Hook/Order: nft nat prerouting (DNAT) auf iifname "ppp*" muss vor filter/forward Enforcement greifen.
- Scope Klarstellung: DNAT betrifft nur eingehenden Client-Traffic auf iifname "ppp*". Server-eigene DNS Queries (OUTPUT) sind nicht Teil dieses DNAT und werden ueber lokale Resolver-Config gesteuert (z.B. resolv.conf -> 127.0.0.1/Unbound/AdGuard).
- WICHTIG (Pfad nach DNAT):
  - Nach DNAT ist dst=10.77.0.1:53 => Traffic läuft über nftables filter input (lokales Ziel), nicht über forward.
- Daher MUSS Restricted Allowlist im input (B150 + docs/nft_service_ip_scopes.txt) explizit UDP+TCP 53 zu 10.77.0.1 erlauben, sonst bricht DNS-Enforcement (insbesondere im Restricted Mode).

7) OPTIONAL
- Kein nftables syslog logging fuer "external DNS attempts" (Noise/Logspam).
- DNS-Transparenz erfolgt ueber AdGuardHome Logs.
- Option B (Nice-to-have, Debug ohne Forensik):
  - Es duerfen Counters (aggregierte Pakete/Bytes) an den DNS-DNAT-Regeln vorhanden sein (UDP/53 und TCP/53 getrennt).
  - Keine Ziel-IPs/Domains/PII erfassen, keine syslog/kernel logs.
  - Zweck: Debug "Sendet der Client ueberhaupt klassisches DNS (Port 53)?" vs. "Problem nach DNAT (Input-Allowlist/DNS-Dienst/Upstream/MTU)".

8) FEHLERBILDER
- Some apps use DoH -> not covered by DNS enforcement (later policy decision).
- IPv6: Dieser Block ist IPv4-only. Konzept-Policy: IPv6 ist serverseitig deaktiviert/gedroppt (Stealth). Falls IPv6 jemals aktiviert wird, muss DNS-Enforcement fuer IPv6 explizit ergaenzt werden (oder IPv6 DNS/Traffic generell droppen), sonst potentieller DNS-Leak ueber v6.
- MTU/Fragmentierung/EDNS0: grosse DNS-Antworten droppen (z.B. DNSSEC/EDNS0) -> DNS wirkt "kaputt", obwohl DNAT korrekt ist. Dann MTU/MSS/Fragmentation prüfen (B090 Kontext).

9) TESTS
- Client set DNS 8.8.8.8 -> still uses 10.77.0.1.
- TCP DNS Test: Client erzeugt DNS über tcp/53 (z.B. große Antwort / erzwungen) -> wird ebenfalls zu 10.77.0.1 umgebogen.
- Restricted Input Test:
  - Client ist restricted (Portal-Only) + setzt DNS manuell auf 8.8.8.8 -> DNS funktioniert trotzdem (DNAT auf 10.77.0.1:53) und input-allowlist lässt tcp/udp 53 zu.
- Negativtest (DoH): Wenn ein Client ausschliesslich DoH nutzt, steigen UDP/TCP-53 Counters nicht (falls Option B aktiv) und DNAT greift nicht. Erwartung: klassisches DNS bleibt erzwungen, DoH ist out-of-scope.
- Optional Counter-Test (wenn Option B aktiv):
  - UDP/53 Counter steigt bei DNS-Queries.
  - TCP/53 Counter steigt bei TCP-DNS-Fallback.
